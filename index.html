Compatibility search

The search is easier now because it supports plain-language prompts, quick filters, and role-based grouping instead of forcing exact item names every time.

Darkstat data

Darkstat already supports static-site generation, API access, item filtering, pinning, hardpoint compatibility search, ship-slot lookup, and Discovery-specific tech compatibility filtering, so your wiki should link to those strengths instead of duplicating the entire tool.

Tech-cell compatibility by ID

Hardpoint-driven search

Darkstat shows internal hardpoints in Ship Details and recommends copying the exact HPType, such as hp_turret_special_6, into Guns, Missiles, Shields, Engines, or other tabs to find compatible equipment.

Use this for: exact slot matching.
Wiki shortcut: a copy-ready hardpoint search guide is easier than making users guess slot names.
Ship slot lookup

Find ships by equipment slot

Darkstat documentation explains that you can search ships by the slot nickname embedded in ship configs, which makes it useful for finding hulls that can mount a specific tool or weapon type.

Weapons

This section keeps the simplified build logic, but now it is written to work as an overview layer above Darkstat’s detailed gun and modifier tabs.

Laser / Photonfast refire

Rapid shield pressure lane

Laser and photon families are the easy recommendation when someone wants fast-firing anti-positron pressure and cleaner hit confirmation in dogfights.

Pulseshield strip

Utility-first shield cracking

Pulse weapons are best treated as a utility group for shield removal before swapping to a hull-focused weapon line.

Plasma

Heavy anti-graviton shots

Plasma rewards precision, offers strong anti-graviton pressure, and suits slower but efficient heavy-shot builds.

Particle

Balanced anti-graviton workhorse

Particle is the easier anti-graviton recommendation when players want better rhythm and usability than plasma.

Tachyon / Neutron

Molecular shield route

Tachyon and neutron families fit molecular-shield hunting, with tachyon leaning range-heavy and neutron leaning efficiency-heavy.

Ships

Ship recommendations are grouped the way players actually choose: strict roleplay, flexible fallback, training, or exact hardpoint hunting.

Faction-bound

Best for strict roleplay

The easiest compatibility rule remains matching your ship line to your faction identity whenever possible.

Universal

Best flexible fallback

Eagle, Saber, and Jackdaw remain the simple fallback names when a faction-specific hull is too weak or too narrow.

Training-friendly

Best for learning combat

Defender and Odin Mark I are easier recommendations for pilots learning snub combat because they sit closer to the middle than extreme specialist ships.

Exact slot path

Best for exact equipment compatibility

When you care more about what a ship can mount than about faction fantasy, use Darkstat’s ship details and HPType search path instead of broad role labels.

Step 1: open Ship Details.
Step 2: copy the exact HPType or weapon slot nickname.
Step 3: search that string in ship or equipment views.

GitHub + Vercel

This starter is intentionally static, so it works well in a GitHub repository right away and can be deployed on Vercel later without changing the core structure.

Recommended repo structure

/index.html for the site entry point.
/data/ later for JSON files like ships, factions, weapons, and compatibility maps.
/assets/ for images, icons, and logos.
/README.md for setup notes and Darkstat source references.

Best next upgrade path

Phase 1: keep this as static HTML on GitHub.
Phase 2: move the compatibility data into JSON files.
Phase 3: add a lightweight frontend on Vercel, such as Vite or Next.js, once the data model is stable.
Phase 4: optionally connect to Darkstat API-backed lookups for live data panels.

Suggested README points

Mention that Darkstat supports static-site generation and can also run as a live backend with API access.
Mention that Darkstat is licensed under AGPL and requires keeping derivative code open and crediting the original author if you fork or modify that code directly.
Clarify that your wiki is a companion layer, not a full replacement for Darkstat’s advanced explorer.

This version is repo-friendly and ready to drop into GitHub as a static site starter before you expand it into a larger Vercel project.
